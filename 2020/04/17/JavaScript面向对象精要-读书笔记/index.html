基本类型和引用类型

1.1什么是类型

原始类型保存为简单的数据值，引用类型保存为对象，其本质是指向内存位置的引用。
为了让开发者能够把原始类型和引用类型按照相同的方式处理（为啥要用相同的方式？）js花费了很大的力气来保证语言的一致性。
其他编程语言用stack存储原始类型，用heap存储引用类型。
js则不同：它用一个变量对象追踪变量的生命周期（原始值被直接保存在变量对象内，而引用值则作为一个指针保存在变量对象内，也就是说大家都保存在变量对象内）

1.2原始类型

boolean,number,string,null,undefined
js和其他语言一样，原始类型的变量直接存储原始值（引用类型存的是内存中的存储位置）

鉴别原始类型

typeof ‘hfskjdhf’ // string
typeof 10 // number
typeof true // boolean
typeof undefined // undefined
typeof null // object empty object point
use value === null // true or false to
console.log(“5” == 5); // true
console.log(“5” === 5); // false
== 会转换类型 ===则不会

原始方法

虽然string,number,boolean是原始类型值，但是他们也有各自的方法(null undefined没有)
‘dfjslf’.toLowerCase();
10.toFixed(2);
true.toString();
‘fhsdkf’.charAt(0)

引用类型指js中的对象，同时也是在js中最接近类的东西。
引用值是引用类型的实例，（请说人话？）
对象是属性的无序列表，属性包含key（key必须是sting）和value
