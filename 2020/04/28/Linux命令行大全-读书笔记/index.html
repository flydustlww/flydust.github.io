3.4符号链接

想象一个这样的场景：一个程序需要使用包含在foo文件中的一个共享资源，但是foo版本变换很频繁，这样，在文件中包含一个版本号会是个好主意。因此管理员就能看到安装了哪个foo版本。这就出现一个问题：如果改变了共享资源的名称，就必须跟踪每一个可能使用了该共享资源的程序，并且当安装了该资源的新版本后，都要让他的程序去寻找新的资源名。
符号链接就解决了这个问题。假设foo的安装版本是2.6，它的文件名是foo-2.6，然后创建一个符号链接指向foo-2.6。这意味着，当一个程序打开foo文件时，他实际上打开的是文件foo-2.6。当需要升级到foo-2.7时，只需将该文件添加到系统里，删除符号链接foo，创建一个指向新版本的符号链接即可。这不仅解决了版本升级的问题，也可以将两种版本都保存在机器里。假如foo-2.7存在一个程序错误，需要切换到旧的版本，同样，只需要删除指向新版本的符号链接，重新创建指向旧版本的符号链接即可

第四章 操作文件与目录

4.1通配符

比如如果想在/usr/bin目录中查找所有以u开始的文件，只需在地址栏输入/usr/bin/u*

4.2 mkdir 创建目录

创建多个目录 mkdir dir1 dir2 dir3

4.3 cp复制文件和目录

cp item1 item2 将单个文件或目录item1复制到文件或目录item2中
选项
-a 复制文件和目录，及其属性。包括权限
-i 在覆盖一个已存在的文件前，提示用户进行确认，如果不加此选项，cp或默认覆盖文件
-r 递归复制目录及内容
u当将文件从一个目录复制到另一个目录时，只会复制哪些目标目录中不存在的文件，或是目标目录相应文件的更新文件

4.4 mv移除和重命名文件

mv可以执行 文件移动 和文件重命名 的操作，完成操作之后，原来的文件将不存在。

4.5 rm 删除文件和目录

unix系统不包含还原删除操作的命令。一旦使用rm命令，就彻底删除了。
rm命令与通配符在一起使用时要格外小心。如下面的例子：
** rm *.html 和 rm * .html 后者会删除当前中所有文件，并且提示目录中没有.html文件
可以使用ls预先对通配符做测试，这将显示要删除的文件，紧接着，你可以用rm代替之前ls，来删除相应的文件

##4.6 ln创建链接

ln命令可以用来创建硬链接或是符号链接。硬：ln file link 符号：ln -s item link
##4.7 硬链接
硬链接是最初unix用来创建链接的方式，符号链接更为先进。默认情况下，每个文件都有一个硬链接，该硬链接会给文件起个名字。当创建一个硬链接的时候，也为这个文件创建了一个额外的目录条目。硬链接有2条重要的局限性：
1 硬链接不能引用自身文件系统之外的文件，也就是说，链接不能引用与该链接不在统一磁盘分区的文件
2 硬链接无法引用目录
